Social Apps

Recovering accurate timelines from chat history and social media downloads — even when metadata is stripped.

The "Chat App" Problem

Most messaging apps and social platforms strip sensitive metadata (EXIF dates, GPS location) the moment you hit send. This protects privacy, but it destroys your timeline when you try to save those photos years later.

PDR solves this by looking where others don't: filenames, embedded JSON data, and file system attributes.

2. Detailed Workflows

WhatsApp

WhatsApp is the most common source of "undated" photos. Thankfully, it saves files with a very specific filename pattern.

The Fix: PDR automatically detects IMG-20190615-WA0002.jpg and understands it was taken on June 15, 2019. It then writes this date back into the file's standard EXIF metadata so other apps can read it.

Facebook & Messenger

Downloading photos one-by-one results in useless filenames like received_987234.jpg. The only reliable way to fix this is a bulk export.

The Fix:

  1. Go to Facebook Settings → "Download Your Information".
  2. Request a download of "Messages" and "Posts".
  3. Choose JSON format (High Quality).
  4. Feed the extracted folder into PDR.

Instagram

Like Facebook, Instagram strips everything. Use the "Download Your Information" tool in Instagram settings.

Tip: Request JSON format if available. PDR scans these exports to rematch photos with their original upload timestamps.

3. Handling "Received" and Random Files

Sometimes you just have a folder full of image5.jpg, download.png, or received_123.jpg with absolutely no metadata or filename clues.

PDR's Last Resort Logic

When all else fails, PDR looks at:

  • File System Creation Date: If the file hasn't been moved across drives too many times, this might still be accurate.
  • Folder Context: If the photo is inside a folder named 2018-Christmas, PDR can use that context.

It ensures that even your messiest folders get sorted into a logical chronological order, rather than being dumped at "Today's Date".
